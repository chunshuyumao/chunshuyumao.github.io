用 VIM 写 Markdown 2.0

前言

昨天《用 VIM 写 Markdown》写的急，字数一多就停了下来，后面再看发现少了些东西。今天特地补充一下。

VIM 中的中文输入法

自己使用的是 Linux 系统，所以只说 Linux 上的输入法。Linux 上常用的输入法有 Ibus（Intelligent Input Bus）和 Fcitx/Fcitx5（Free Chinese Input Toy of X）。

Ibus 和 Fcitx5 没有太大的区别，无非是顺不顺手罢了。我用的桌面协议是 Wayland，据说 Fcitx5 对 Wayland 的支持比较好、不是很卡顿，因此我一直使用的是 Fcitx5。解决在 VIM 中的中文输入法也是以 Fcitx5 为例子。

VIM 有多种模式，模式的切换使用的都是英文键盘。当自己正在疯狂打字输出的时候突然切换模式（按 Esc 键），VIM 收到命令会退回正常模式（Normal Mode）。这时候进行其他操作，例如进入命令行模式处理文本或者移动光标，此时键盘仍是中文键盘，VIM 识别不了命令，想要让 VIM 正确识别命令需要手动切换键盘。

我们可以让 VIM 在退出插入模式（Insert Mode）或者在进入 VIM 时自动切换到英文键盘，减少手动的次数。

Markdown 换行问题

上一篇文章说到，可以使用回车符把长行分成多段，然后用 Pandoc 的 east_asian_line_breaks 处理 Markdown。不知道有没有人发现，我昨天的公众号文章有些中文莫名其妙出现空格？那是我使用的公众号 Markdown 格式渲染工具不支持类似 Pandoc 自动识别东亚文字的功能。

Markdown 会把回车符分开的段落合并成一行，并在它们之间添加空格 — 所有的 Markdown 语言都支持这个操作，只是 Pandoc 还支持自动识别东亚文字，然后去掉空格。我使用的公众号渲染工具没有 Pandoc 的类似功能，识别不了东亚文字，于是它自动给合并的段落添加了空格，文章看起来就怪怪的。

这就引申出了另一个问题：如果我写 Markdown，为了不让段落太长进行了分段，是不是只能使用 Pandoc 进行渲染，用其他的工具仍然有东亚文字出现空格的现象？

这个问题解决办法很简单，还是通过 Pandoc 来处理：只需要设置 Markdown 的导出格式为 Markdown 即可。这是很容易忽略的点：很多人只知道 Pandoc 可以从格式一转换到格式二，但是忘了也可以从格式一转换到格式一。
