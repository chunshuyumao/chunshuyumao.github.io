Tmux

Tmux（Terminal Multiplexer）是一个 Linux 终端复用器。根据官网的介绍， “它可以让使用者在一个终端内轻松切换多个程序，分离终端（后台运行），和重新连接（已经分离的）终端”。

听不懂没关系，简单来说就是以下几个特点：

  1. 终端分屏
  2. 终端分身
  3. 后台运行

终端分屏，现在很多比较高级的 Linux 终端都配备，比如 Terminator 。不过这些都算是外力，对于没有桌面或者没有管理员权限的 Linux 系统来说，想要安装 Terminator 是不可能的。Tmux 可以作为一个可执行软件直接放到用户目录下就能完成它的使命。

终端分身，这是我生造的词。其实就是 Tmux 官网上说的，可以在一个终端内运行、切换多个程序。在某程度上和“分屏”是一个意思，但是“分屏”暗含一个窗口的意思，“分身”则表明不仅可以有一个窗口。

后台运行，就是“不挂起”（no hang up）。有些时候使用命令行下载或者执行某个命令需要的时间比较长，我们就会选择把命令放到后台运行但不挂起，俗称”后台运行“。

基础用法

打开一个终端，然后键入 tmux 并回车就可以进入 Tmux 了。

进入 Tmux 后可以看到终端底部多了一段状态栏，除此之外，它和你的普通终端没有任何区别。

首先介绍一下 Tmux 的快捷键。Tmux 所有的快捷键都有一个“引导键”（prefix），默认是 ctrl + b，但可以更改。由于每个人可能使用不同的引导键，出于习惯我会使用 <prefix> 代替引导键。后面见到 <prefix> 默认是 ctrl + b 或者自己的引导键。

举个例子，如果我表示键入 <prefix>a，则表示按下 ctrl + b + a。依次按下即可，不需要以迅雷不及掩耳之势快速同时毫无延时地按下 --- 没有哪个程序那么变态。

分屏

  • 左右分屏：<prefix>%
  • 上下分屏：<prefix>"

<prefix>%，是按 ctrl + b + shift + 5，因为 % 在主键盘（不是数字键盘） 5 的位置。同理 <prefix>" 是 ctrl + b + shift + '，单引号。

创建窗口

  • 创建窗口：<prefix>c

下图是 Tmux 的结构，启动 Tmux 会自动创建一个会话（session），会话里可以创建多个窗口（window），每个窗口又可以通过分屏（split）创建多个窗格（pane）。

创建窗口的快捷键是 <prefix>c，c 是 create 的缩写。

创建窗口后可以通过 <prefix>w 查看所有窗口，w 是 window 的缩写，同时借助方向键和回车键 Enter 选择窗口。

脱离会话

  • 脱离会话：<prefix>d

d 是 detach 的缩写。控制台运行的程序在控制台退出（exit）后都会自动销毁，所以 Linux 终端后台运行一般使用 nohup <command> &。

使用了 Tmux 就不用这么麻烦，可以通过快捷键脱离（detach）会话，Tmux 会在 /tmp 目录保留一个通信文件，位于 /tmp/tmux-xxxx 目录下，xxxx 代表数字。除非电脑除非电脑重启或者关机，否则这个会话会一直驻留后台。

脱离会话之后，在 Tmux 中运行的程序不会自动终止，而是继续运行，所以起到和后台运行一样的效果。

那如何结束会话？注意，是结束会话，不是脱离会话。结束会话可以直接在终端输入 exit 即可。

重新连接会话

  • 重连会话：tmux attach -t <id>

上面的命令是在终端输入的，<id> 是会话的数字。Tmux 可以创建多个会话，所以需要使用 -t 指定连接的会话。比如我的电脑一般都只会启用一个会话，所以重连的时候只需要使用 tmux attach -t 0 即可。

Tmux 的用处

保持服务器连接，避免任务中断

上面简单介绍了 Tmux 的操作，但没说它的作用。Tmux 的一个用处是：服务器连接。一般在服务器操作都需要使用 SSH 登陆服务器。

SSH 默认一段时间不操作就会断开，这是为了服务器的安全着想。有些管理人员为了避免登陆服务器的人频繁重连，会把 SSH 闲置（IDL）时间设置长一点。

不过这是治标不治本，如果下载的东西花的时间很久，SSH 还是会断开，然后得重新登陆，重新下载，周而复始。一个办法是使用 nohup 后台运行下载程序。不过后台运行是没有输出信息的，所以使用 nohup 又要查看输出信息的话，一般需要各种输出重定向，非常麻烦。

使用 Tmux，就算真的长时间不操作，服务器自动断开了连接，也不需要担心自己的命令终止，因为这相当于自动脱离了 Tmux 会话。只要后面登陆服务器，就可以再次连接 Tmux 会话，而且所有的输出信息都会保留作为记录。

复用 SSH 会话，减少连接数

Linux 会限制同一地址的 SSH 连接数，默认是 10 个连接。也就是说如果使用普通的终端直接连接 Linux 服务器，最多只能开 10 个标签（Tab）。

然而如果使用 Tmux，理论上 Tmux 可以创建任意多的会话和窗口，但它只会占用一个连接数，而且还不需要重新输入密码。

打开另一个终端标签重新连接服务器，需要重新输入密码。如果使用 Tmux，它会复用当前的连接，所以不需要输入密码。
